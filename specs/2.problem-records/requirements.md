# problem-records — 需求规格

## 概述

题目记录的完整 CRUD：新增/编辑/删除题目，支持按标签、难度、是否一次AC筛选，提供详情页查看笔记。

## 项目信息

- 项目名: algorithm-tracker
- 架构类型: Turborepo monorepo

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-17 | v1   | 初始需求 |

## 用户故事

- 作为用户，我想要记录每道题的做题信息，以便日后复盘。
- 作为用户，我想要按标签/难度/是否AC筛选题目，以便快速找到特定类型的题。
- 作为用户，我想要查看题目详情和笔记，以便回顾当时的思路。

## 功能需求

1. [F-001] 题目字段：题目名称（必填）、来源（可选，文本）、难度（简单/中等/困难，必填）、标签（多个，可选）、做题日期（必填，默认今天）、用时（分钟，整数，可选）、是否一次AC（布尔，默认 false）、笔记（Markdown 文本，可选）
2. [F-002] 支持新增题目（表单页）
3. [F-003] 支持编辑题目（复用同一表单，回填数据）
4. [F-004] 支持删除题目（二次确认）
5. [F-005] 题目列表页：卡片展示，显示标题、难度、标签、日期、是否AC
6. [F-006] 列表支持按标签（单选）、难度（单选）、是否AC（三态：全部/已AC/未AC）筛选
7. [F-007] 题目详情页：展示全部字段，笔记区域渲染 Markdown

## 非功能需求

- 性能: 列表页在 100 条记录内加载 < 1s
- 安全: 所有 tRPC 操作使用 `protectedProcedure`，确保登录后才可访问

## 验收标准

- [ ] [AC-001] 能成功新增一条题目记录，所有字段正确保存到 DB
- [ ] [AC-002] 编辑后字段更新，标签变更正确同步（增删）
- [ ] [AC-003] 删除后记录从列表消失，DB 中不存在
- [ ] [AC-004] 筛选：按难度"中等"过滤只显示 difficulty=medium 的记录
- [ ] [AC-005] 筛选：按标签"数组"过滤只显示含该标签的记录
- [ ] [AC-006] 详情页笔记区域正确渲染 Markdown（如 `**加粗**` 渲染为粗体）
