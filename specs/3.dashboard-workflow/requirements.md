# dashboard-workflow — 需求规格

## 概述

统计看板展示刷题总量、本周数量、连续打卡天数和近30天趋势图；/workflow 页展示 AI 辅助开发流程的 Mermaid 可视化。

## 项目信息

- 项目名: algorithm-tracker
- 架构类型: Turborepo monorepo

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-17 | v1   | 初始需求 |

## 用户故事

- 作为用户，我想要在首页看到总刷题数、本周刷题数、连续打卡天数，以便直观了解刷题节奏。
- 作为用户，我想要看到近30天每日刷题数量的折线图，以便发现刷题节奏的规律。
- 作为用户，我想要在 /workflow 页看到 Mermaid 图，以便满足课程作业展示要求。

## 功能需求

1. [F-001] 统计卡片：总刷题数（problems 表总行数）
2. [F-002] 统计卡片：本周刷题数（当前自然周 Mon–Sun 内的 date 字段记录数）
3. [F-003] 统计卡片：连续打卡天数（从今天向过去数，连续有题目记录的天数；当天无记录则从昨天起算）
4. [F-004] 近30天趋势折线图：X轴为日期，Y轴为当天题目数，无数据的天显示0
5. [F-005] /workflow 页：静态页面，包含一张 Mermaid 流程图，展示 init→prd→ai 三阶段的 AI 辅助开发流程，无需交互

## 非功能需求

- 性能: 统计接口聚合查询 < 500ms（个人数据量小）
- 兼容性: Mermaid 图在 Chrome 最新版正常渲染

## 验收标准

- [ ] [AC-001] 新增一条今日题目记录后，Dashboard 总数+1、本周数+1、连续打卡正确更新
- [ ] [AC-002] 中断打卡后连续天数归零（今日和昨日均无记录时为0）
- [ ] [AC-003] 折线图正确渲染30天数据，无记录的天显示为0
- [ ] [AC-004] /workflow 页 Mermaid 图正常渲染，无控制台报错

## 依赖

- Feature 2（2.problem-records）：problems 表存在，有数据
- `recharts`（折线图）
- `mermaid`（流程图渲染）
