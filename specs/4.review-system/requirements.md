# review-system — 需求规格

## 概述

错题本系统：用户手动标记题目"需要二刷"，系统按艾宾浩斯曲线（第1/3/7/15/30天）生成5次复习提醒，5次全部完成后归档。

## 项目信息

- 项目名: algorithm-tracker
- 架构类型: Turborepo monorepo

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-17 | v1   | 初始需求 |

## 用户故事

- 作为用户，我想要在题目详情页标记"需要二刷"，以便加入错题复习队列。
- 作为用户，我想要在错题本页看到今日及过期待复习题目，以便按计划复习。
- 作为用户，我想要点击"已复习"完成一次复习，以便记录进度并更新下次提醒日期。

## 功能需求

1. [F-001] 题目详情页/列表卡片提供"标记二刷"按钮，点击后该题进入错题本
2. [F-002] 标记时自动计算5个复习日期：标记日 + 1/3/7/15/30 天
3. [F-003] 错题本页（/review）展示"待复习"列表：`next_review_at ≤ 今天` 且未归档
4. [F-004] 每条待复习条目显示：题目标题、难度、标签、第N次复习（N/5）、原定复习日期
5. [F-005] 点击"已复习"：复习次数+1，若达到5次则归档（移出待复习列表），否则更新下次复习日期为曲线下一节点
6. [F-006] 已归档的题目不再出现在待复习列表（可在题目详情页查看归档状态）
7. [F-007] 同一题目重复点击"标记二刷"：若已在复习中则提示"已在复习队列"；若已归档则重置复习进度、重新开始5次计划

## 非功能需求

- 安全: 所有 review 操作使用 `protectedProcedure`

## 验收标准

- [ ] [AC-001] 标记后，DB 中 `needs_review=true`，`next_review_at` 为标记日+1天
- [ ] [AC-002] 点击"已复习"第1次，`review_count=1`，`next_review_at` 更新为标记日+3天
- [ ] [AC-003] 第5次复习完成后，`review_archived=true`，条目从待复习列表消失
- [ ] [AC-004] 待复习列表只显示 `next_review_at ≤ 今天 AND review_archived=false` 的记录
